URL - http://validator.w3.org/
Content Description- Has opening head information with meta tags, a title and links. A body with div's id's ul's li's p's a's and a longer list of elements I do not know.
Validation- Did not pass validator.nu. Passed HTML Lint Validation.
Errors- There were 5 errors on validator.nu. There were no errors using HTML Lint Validation.
Solutions- Validator.nu errors are only prevalent because it is not written with HTML5 standards. There were no errors using HTML Lint Validation.
URL - http://lint.brihten.com/html/
Content Description- Has opening head information with meta tags, title and links. Body has class, div, id, h1, ul, li, h3, h2, h4, p, dt, dd and dl.
Validation- Does not pass validator.nu. Does pass on HTML Lint Validation.
Errors- Validator.nu. has two errors. HTML Lint Validation has none.
Solutions- Both validator.nu errors are from placing a div inside of a button. This is impossible because buttons are inline elements and divs are block elements, inline elements cannot hold block elements. So taking the div out of the inline element would solve the problem.
