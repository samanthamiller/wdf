Checking HTML5 of four Websites

Websites Checked

  1. Vouge UK-

    Vouge's content includes photos, current events, navigation, blogs, external and internal links. They do not use HTML5 properly, they have headers that are not labeled and not enough sections to make the outline make since and follow their site structure. To fix errors they could add labels to their headings, add sections to label their subheadings and articles for their RSS feed as well as making sure their code is in the semantic order of the web page.

  2. Keiyasui-

    Keiyasui's content includes photos, navigation, external and internal links. For the most part the use HTML5 properly. There are a couple "Untitled Section"s which as far as I can tell are simple because it is a nav function which does not need a title per say.

  3. Codeman-

    Codemans's content includes photos, navigation, external and internal links. For being the "codeman" his HTML5 structure is lacking, there is none. When ran through the outline only one thing pops up and it pops up as an "Untitled Section". To fix these errors he should implement everything in the HTML5 structure.

  4. Shell Heroes-

    Shell Hero's content includes a background and a game platform. In the outline only one "Untitled Section" shows up. This is the header section being called out which is appropriate because there is no need to name the header being that all the websites content is in the game platform which uses another language to host it.
